Material

O3DE 的材质大致由四个相互关联的文件组成：

  • .azsl：着色器代码。
  • .shader：JSON 格式，持有一个 .azsl 引用，对渲染管线的配置。
  • .materialtype：JSON 格式，持有一个或多个 .shader 引用，绑定材质中的属性与 shader 中的变量。
  • .material：JSON 格式，持有一个 .materialtype 的引用，由 Material Editor 生成。

示例

.azsl

包括：

  • include 着色器资源组
  • Vertex Shader 和 Fragment Shader 输入输出的定义
  • Vertex Shader 和 Fragment Shader 的代码

.shader

包括：

  • 一个 .azsl 文件的引用
  • 配置 depth, stencil, blend 状态
  • 配置 Shader 编译器 hint
  • 指定 Vertex Shader 和 Fragment Shader 程序的入口点
  • 指定该 Shader 应该加入哪个渲染队列

.materialtype

包括：

  • 一个或多个 Shader 的引用
  • 定义暴露给编辑器的属性/属性组
  • 绑定属性和 Shader 中的变量

或者可以看看官方提供的标准 PBR 材质类型：
StandardPBR.materialtype

.material

.material 可以被模型引用，也可以在编译器里手动分配给模型。
在运行时，.material 可以在内存中存在多个实例。

Assimp

Assimp 中的 material.h

枚举

  • aiTextureOp
    两张贴图之间的结合方式

  • aiTextureMapMode
    超出范围的 uv 坐标如何 map

  • aiTextureMapping
    uv 的应用方式，SPHERE/BOX

  • aiTextureType
    贴图类型，从布林冯到 PBR 都有

  • aiShadingMode
    渲染方式，从布林冯到 PBR 都有

  • aiTextureFlags
    是否使用 alpha 通道

  • aiBlendMode
    透明的混合方式

  • aiUVTransform
    uv 的变换

  • aiPropertyTypeInfo
    简单的 RTTI，存储 property 实际的数据类型

aiMaterialProperty

assimp 建议通过 aiMaterial::Get 而非该结构体访问他们

aiMaterial

  • Get
    Get 的定义类似

调用时第一个参数必须为 AI_MATKEY_XXX，这个宏直接包含了前三个参数

  • GetName
    返回该 material 的名字

  • GetTextureCount
    返回指定 Texture Type 的数量

  • GetTexture
    返回指定 type 指定 index 的贴图路径，index 由 GetTextureCount 得到

  • AddProperty
    第三个参数依旧是 AI_MATKEY_XXX

  • RemoveProperty
    对应于 AddProperty
