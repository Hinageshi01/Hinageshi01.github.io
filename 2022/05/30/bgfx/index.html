笔记 bgfx

本文最后更新于：June 1, 2022

序言

  bgfx 是一个跨平台的渲染框架。它将 DX、GL、VK 等图形 API 封装成了统一的接口，抹除了这些 API 之间的差异性。这样我们在写渲染逻辑的时候就只需要调用这层 RHI 提供的 API 就好，而不需要关心程序实际上使用的是什么图形 API 了。

bgfx 的使用通常可以从实现 entry::AppI 接口中

  • virtual void init(int32_t _argc, const char* const* _argv, uint32_t _width, uint32_t _height)
  • virtual int shutdown()
  • virtual bool update()

这三个纯虚函数开始。

init

bgfx 初始化资源是用 bgfx::Init 存储初始化信息，通过 bgfx::init(init) 接口进行初始化的。

bgfx::init() 的声明

该函数会在设置了初始化参数之后调用 bool Context::init(const Init& _init) 完成真正的初始化工作，这之后是一些错误处理工作。

PlatformData 结构体的声明

所有成员变量都被声明为 void* 类型，以便接受各个平台各类型的数据对象。
所有成员变量都被默认初始化为 NULL，此时 bgfx 会负责创建对应的对象。

bgfx.cpp 中含有一个全局的 bgfx::g_platformData 变量，可以通过 void setPlatformData(const PlatformData& _data); 进行设置
