随记

  • functions.h 提供了所有函数的声明，真正的实现在 functions.glsl 里，cpp 版的实现在 functions.cc 里定义了几个宏之后单纯 include 了 functions.glsl。
  • 工程分为两个部分，CPU 版和 GPU 版，CPU 版主要用于验证 GPU 版中的各类近似（半精度浮点，从单个波长反推 single Mie scattering，只使用 RGB 三个波长计算亮度值而非 48 个波长）。
  • 在 cpp 版的 definitions.h 中用 typedef 对不同物理量进行了区分，以防产生不合理的转换，但在 GLSL 版的 definitions.glsl 中没有 typedef，所有物理量都简单 define 成 float 或 vec3 了…
  • 在 GeomotryShader 中用 uniform int layer; 和 gl_Layer = layer; 指定 3DTexture 绘制到哪一层。
  • 首先将 r, mu 映射至 [0, 1]，然后将 f(0) 存在第 0 个像素，将 f(1) 存在第 n-1 个像素。
  • num_precomputed_wavelengths 如果小于等于三，散射则被计算为三个波长并存储为 irradiance，否则会被四舍五入至三的倍数并存储为 illuminance。
  • model 的构造函数里搞了一堆 lambda 去生成 string 拼凑成 shader…
  • 使用 irradiance 模式进行渲染时，不需要 radiance 和 luminance 之间的转换，此时 luminance_from_radiance 为单位矩阵。
  • Precomputed Single Scattering 不包含相函数，相函数在运行时添加，以获得更好的角度精度。
  • 如果地平线出现了紫色/黄色条带，多半是 Transmittance 精度问题。
  • 如果黄昏时天色没有变暗变红，多半也是 Transmittance 的问题。
